## Опис проєкту

Інтелектуальний агент персонального асистента, реалізований як робочий процес платформи автоматизації **n8n**. Агент взаємодіє з користувачем через Telegram, приймає запити в текстовій, голосовій, фотографічній та документній формах, відповідає природною мовою з урахуванням попереднього контексту взаємодії та публікує згенерований контент у комунікаційні канали.

**Ключові можливості:**

- Мультимодальний ввід: текст, голос (Whisper), зображення (GPT Vision), документи (PDF, DOCX, TXT);
- Трирівнева пам'ять: Redis (оперативна), ZEP thread (сесійна), ZEP knowledge graph (довготривала з темпоральними мітками);
- Виклик функцій (function calling) через LangChain AI Agent для пошуку та збереження фактів про користувача;
- Маркерна система постобробки: `[IMAGE: ...]`, `[IMAGE_EDIT: ...]`, `[PUBLISH]`;
- Генерування зображень через gpt-image-1 (OpenAI);
- Публікація в Telegram-канали з опційним підтвердженням (Human-in-the-Loop);
- Журналювання операцій у Google Sheets;
- Команди керування: `/settings`, `/memory`, `/reset`.

Загальна кількість виконавчих вузлів у робочому процесі — **93**, об'єднаних у шість функціональних підсистем (взаємодія, оркестрація, інтелектуальне ядро, пам'ять, обробка медіа, зовнішні інтеграції).

## Технологічний стек

- **n8n** — платформа оркестрації робочих процесів;
- **OpenAI API** — мовна модель сімейства GPT, Whisper (STT), gpt-image-1 (генерування зображень);
- **ZEP** — темпоральний граф знань для довготривалої пам'яті агента;
- **Redis** — оперативне сховище історії розмов та налаштувань користувача;
- **Telegram Bot API** — канал взаємодії з користувачем;
- **Google Sheets API** — журналювання операцій;
- **LangChain** — фреймворк AI Agent для function calling.

## Розгортання

### Варіант А (основний) — n8n Cloud

Цей сценарій рекомендований для членів екзаменаційної комісії: не потребує локального встановлення, готовий за 10–15 хвилин.

**Крок 1. Створення облікового запису n8n Cloud**

1. Відкрити <https://n8n.cloud> → **Get started**.
2. Зареєструватися на тарифному плані **Starter** (від 20 EUR/міс, доступний 14-денний тріал).
3. Після створення робочого простору буде доступний URL виду `https://<your-instance>.app.n8n.cloud`.

**Крок 2. Імпорт workflow**

1. У n8n Cloud відкрити **Workflows → Add workflow → Import from File**.
2. Завантажити файл `AI Personal Assistant — БКР Дудинець v11.4.json` із цього репозиторію.
3. Натиснути **Save**.

**Крок 3. Налаштування credentials**

У n8n Cloud у розділі **Credentials → Add Credential** створити по одному запису для кожного сервісу:

- **OpenAI** → тип `OpenAI API`, поле `API Key` — ключ з OpenAI;
- **Telegram Bot** → тип `Telegram API`, поле `Access Token` — токен від BotFather;
- **ZEP** → тип `HTTP Header Auth`, заголовок `Authorization`, значення `Api-Key <ваш ZEP API key>`;
- **Redis** → тип `Redis`. Для n8n Cloud рекомендується хмарний Redis (наприклад, [Redis Cloud](https://redis.com/try-free/) з free tier 30 МБ): вписати host/port/password з консолі Redis Cloud;
- **Google Sheets** *(опційно)* → тип `Google Sheets OAuth2 API`, пройти OAuth-авторизацію акаунта Google.

Після створення credentials у workflow на полотні n8n кожен вузол (Telegram Trigger, OpenAI Chat Model, ZEP-вузли, Redis Chat Memory, Log to Google Sheets) **відкрити по черзі**, у полі `Credentials` обрати щойно створений запис, **зберегти**.

**Крок 4. Конфігурація допустимих користувачів (Owner Check)**

У вузлах `Owner Check` та `Callback Owner Check` (IF-тип) у полі умови вписати свій Telegram `userId`, отриманий через [@getmyid_bot](https://t.me/getmyid_bot). За замовчуванням у JSON стоїть `userId` автора — його потрібно замінити на власний, інакше бот відмовлятиме у доступі.

**Крок 5. Активація**

1. У верхньому правому куті полотна натиснути перемикач **Active**.
2. У Telegram знайти свого бота за іменем, надіслати команду `/start`.
3. Надіслати тестове повідомлення — бот має відповісти текстом.

### Варіант Б (додатковий) — Self-hosted у Docker

Цей сценарій підходить для повного локального розгортання без оплати n8n Cloud. Усі компоненти (n8n, Redis) запускаються у Docker.

**Крок 1. Встановити Docker Desktop**

<https://www.docker.com/products/docker-desktop> → встановити для Windows / macOS / Linux. Після встановлення переконатися, що `docker --version` повертає номер версії в терміналі.

**Крок 2. Запустити n8n + Redis**

Створити файл `docker-compose.yml` з таким вмістом:

```yaml
services:
  n8n:
    image: docker.n8n.io/n8nio/n8n
    container_name: n8n
    ports:
      - "5678:5678"
    environment:
      - N8N_SECURE_COOKIE=false
    volumes:
      - n8n_data:/home/node/.n8n

  redis:
    image: redis:7-alpine
    container_name: redis
    ports:
      - "6379:6379"
    volumes:
      - redis_data:/data

volumes:
  n8n_data:
  redis_data:
```

Запустити:

```bash
docker compose up -d
```

Відкрити <http://localhost:5678>, створити обліковий запис власника робочого простору.

**Крок 3–5.** Імпорт workflow, налаштування credentials, конфігурація Owner Check і активація — **ті самі що у Варіанті А**.

Відмінність лише в Redis credential: для self-hosted host = `redis` (службове ім'я з docker-compose), port = `6379`, пароль не потрібен.

## Перший тест бота

Після активації workflow надіслати боту в Telegram:

1. **Текст**: «Привіт!» → бот відповідає природною мовою;
2. **Голосове повідомлення**: будь-яка фраза → бот розпізнає через Whisper і відповідає;
3. **Фото з підписом**: «Що на цьому фото?» → бот аналізує зображення через GPT Vision;
4. **Запит на зображення**: «Намалюй кота за столом» → бот генерує і надсилає картинку;
5. **Команда `/memory`** → бот повертає список збережених фактів про користувача;
6. **Команда `/settings`** → інлайн-меню з налаштуваннями HITL і списку каналів для публікації.
