**Тема роботи:** Розроблення інтелектуального агента персонального асистента з функціями відповіді на запитання та генерації контенту.

## Опис проєкту

Інтелектуальний агент персонального асистента, реалізований як робочий процес платформи автоматизації **n8n**. Агент взаємодіє з користувачем через Telegram-бот, приймає запити в текстовій, голосовій, фотографічній та документній формах, відповідає природною мовою з урахуванням попереднього контексту взаємодії та публікує згенерований контент у комунікаційні канали.

**Ключові можливості:**

- Мультимодальний ввід: текст, голос (Whisper), зображення (GPT Vision), документи (PDF, DOCX, TXT);
- Трирівнева пам'ять: Redis (оперативна), ZEP thread (сесійна), ZEP knowledge graph (довготривала з темпоральними мітками);
- Виклик функцій (function calling) через LangChain AI Agent для пошуку та збереження фактів про користувача;
- Маркерна система постобробки: `[IMAGE: ...]`, `[IMAGE_EDIT: ...]`, `[PUBLISH]`;
- Генерація зображень через gpt-image-1 (OpenAI);
- Публікація в Telegram-канали з опційним підтвердженням (Human-in-the-Loop);
- Журналювання операцій у Google Sheets;
- Команди керування: `/settings`, `/memory`, `/reset`.

Загальна кількість виконавчих вузлів у робочому процесі — **93**, об'єднаних у шість функціональних підсистем (взаємодія, оркестрація, інтелектуальне ядро, пам'ять, обробка медіа, зовнішні інтеграції).

---

## Технологічний стек

- **n8n** — платформа оркестрації робочих процесів (low-code, self-host);
- **OpenAI API** — мовна модель сімейства GPT, Whisper (STT), gpt-image-1 (генерування зображень);
- **ZEP** — темпоральний граф знань для довготривалої пам'яті агента;
- **Redis** — оперативне сховище історії розмов та налаштувань користувача;
- **Telegram Bot API** — канал взаємодії з користувачем;
- **Google Sheets API** — журналювання операцій;
- **LangChain** — фреймворк AI Agent для function calling.

## Передумови

Для розгортання потрібно встановити:

- **Node.js 18+** та **Docker Desktop** (рекомендовано);
- Облікові записи та API-ключі:
  - **OpenAI API** — ключ доступу до API мовної моделі ([platform.openai.com](https://platform.openai.com));
  - **Telegram Bot Token** — створюється через [@BotFather](https://t.me/BotFather);
  - **ZEP Cloud API key** — реєстрація на [getzep.com](https://www.getzep.com);
  - **Google Sheets API** — credentials JSON (опційно, для журналу);
  - **Redis** — локальний інстанс або хмарний (можна Redis Cloud).

## Встановлення та запуск

### Крок 1. Розгортання n8n (Docker)

```bash
docker run -it --rm \
  --name n8n \
  -p 5678:5678 \
  -v ~/.n8n:/home/node/.n8n \
  docker.n8n.io/n8nio/n8n
```

Після запуску відкрити в браузері: <http://localhost:5678> і створити обліковий запис власника.

### Крок 2. Розгортання Redis

```bash
docker run -d --name redis -p 6379:6379 redis:7-alpine
```

### Крок 3. Імпорт workflow

1. У n8n натиснути **Workflows → Import from File**.
2. Обрати файл `AI Personal Assistant — БКР Дудинець v11.4.json` з цього репозиторію.
3. Натиснути **Import**.

### Крок 4. Налаштування credentials

У n8n у розділі **Credentials → Add Credential** додати:

- **OpenAI API** — Bearer Token з ключем OpenAI;
- **Telegram API** — Bot Token від BotFather;
- **HTTP Header Auth (ZEP)** — Header `Authorization: Api-Key <ваш ZEP API key>`;
- **Redis** — host: `localhost`, port: `6379`, без пароля для локальної інсталяції;
- **Google Sheets OAuth2** — (опційно) для запису журналу операцій.

У кожному вузлі workflow, що використовує цей credential, обрати відповідний запис.

### Крок 5. Конфігурація допустимих користувачів

У вузлах `Owner Check` та `Callback Owner Check` встановити власний Telegram `userId` у полі IF-умови (отримати свій `userId` можна через будь-якого [getIDsBot](https://t.me/getmyid_bot)).

### Крок 6. Активація

1. Активувати workflow перемикачем у верхньому правому куті полотна.
2. Знайти бота в Telegram за іменем (вказаним при створенні).
3. Надіслати тестове повідомлення.
